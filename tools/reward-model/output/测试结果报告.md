## 二、单笔订单分析（最坏情况）

### 2.1 普通用户单笔订单

```
成本:         ¥80.00
售价:         ¥104.00   (80 × 1.30)
利润:         ¥24.00

利润去向:
  平台利润(50%):      +¥12.00   ← 确定收入
  慈善+科技+备用(18%): +¥4.32    ← 平台可控
  卖家产业基金(16%):   -¥3.84    ← 真实支出
  奖励流出(16%):       -¥3.84    ← 最坏=全部提现
  ─────────────────────────────
  平台单笔净收入:      +¥12.48   ✓ 占售价12.0%
```

### 2.2 VIP用户单笔订单（rebateRatio=0.5，默认）

```
成本:         ¥120.00
售价:         ¥148.20   (120 × 1.30 × 0.95折扣)
利润:         ¥28.20

利润去向:
  非rebate留平台(50%利润): +¥14.10
  rebate中平台+基金(40%):  +¥5.64
  奖励流出(rebate×60%):    -¥8.46    ← 最坏=全部提现
  ─────────────────────────────
  平台单笔净收入:          +¥11.28   ✓ 占售价7.6%
```

### 2.3 VIP用户单笔订单（rebateRatio=1.0，前期激进）

```
成本:         ¥120.00
售价:         ¥148.20
利润:         ¥28.20

利润去向:
  rebate = 100%利润:      ¥28.20
  平台+基金(40%):          +¥11.28
  奖励流出(60%):           -¥16.92   ← 最坏=全部提现
  ─────────────────────────────
  平台单笔净收入:          -¥5.64    ✗ 单笔亏损！占售价-3.8%
```

> **结论**：rebateRatio=1.0时VIP单笔订单亏损。这是唯一会导致单笔亏损的参数配置。

## 七、风险提示

### 🔴 高风险

1. **rebateRatio=1.0 导致VIP单笔亏损** — 前期激进配置需谨慎，建议不超过0.8
2. **抽奖成本是最大支出** — ¥270,000/月，超过分润流出的1.75倍

### 🟡 中风险

3. **用户购买频率低于5次/月时，含抽奖情况下最坏情况亏损** — 需要抽奖成本配合调整
4. **加价率低于1.35时，最坏情况下含抽奖会亏损** — 建议加价率不低于1.35

### 🟢 低风险

5. **分润系统本身** — 任何参数组合下，分润系统的净贡献都是正的
6. **树结构参数（叉数/层数）** — 对盈利影响极小

---

## 八、下一步建议

1. **优先调整抽奖参数**：降低日活跃率(30%→15%)、中奖率(60%→40%)、或单次成本(5→3元)，可将抽奖成本从27万降至5-8万
2. **加价率建议≥1.35**：在最坏情况下也能保证盈利
3. **rebateRatio建议≤0.8**：避免VIP单笔亏损
4. **运行第二层时序仿真**：验证树动态、早期用户集中度、现金流峰值
