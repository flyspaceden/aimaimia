# 国内联调指南（不依赖 Expo Go / Tunnel）

> 目标：在中国也能稳定开发/联调本项目。核心原则：不依赖 Expo 的线上服务（tunnel/扫码/云构建），只走本地 Metro + 局域网/USB/Dev Client。

---

## 1) 你需要什么环境

### 1.1 Node / npm
- 建议使用 LTS（以项目 `package.json` 为准）。
- 安装依赖：`npm i`

### 1.2 真机/模拟器
- **Android 推荐**：更容易在国内稳定联调（USB + adb reverse）。
- iOS 真机：需要 Mac + Xcode（本地运行/Dev Client 也需要）。

---

## 2) 本地开发方式（优先级从高到低）

### 方案 A（推荐）：Android USB + `adb reverse`（最稳，不受 Wi‑Fi 影响）
1) 手机打开开发者选项，启用 USB 调试
2) 连接电脑后执行：
   - `adb devices`
   - `adb reverse tcp:8081 tcp:8081`
3) 启动 Metro：
   - `npx expo start --localhost`
4) 打开 App（Dev Client 或本地原生构建），即可加载本地 Metro。

> 说明：此方式不需要 tunnel，不依赖外网，也不需要同一 Wi‑Fi。

### 方案 B：同一局域网（LAN）
1) 电脑和手机在同一 Wi‑Fi
2) 启动 Metro：
   - `npx expo start --lan`
3) 如果你在使用 Dev Client，可在 App 内输入 Metro 地址或扫码（仅局域网二维码）。

### 方案 C：Dev Client（后续接高德地图/支付等原生 SDK 必备）
适用场景：
- 不想依赖 Expo Go
- 需要接入原生 SDK（高德地图/微信支付/支付宝/推送等）

基本思路：
- 通过 `expo prebuild` 生成 `ios/`、`android/` 原生工程
- 构建一个“开发版 App”（Dev Client）
- Dev Client 安装到手机后，连接本地 Metro 热更新

> 备注：后续如果接入高德地图原生 SDK，必须走 Dev Client/原生工程；Expo Go 不支持。

---

## 3) 常见问题排查

### 3.1 手机能打开 App，但加载不出页面
- 优先检查是否使用了 `--tunnel`（国内可能不稳定）。推荐改用方案 A/B。
- 若是 Android：优先使用 `adb reverse` + `--localhost`。

### 3.2 图片大量空白
- 当前 Mock 数据里使用了 `https://placehold.co/...` 占位图，国内可能不稳定。
- 建议后续替换为：
  - 本地 `assets/` 图片
  - 或国内 CDN（例如阿里云/腾讯云）

### 3.3 需要接入高德地图/微信支付/推送
- 这些属于原生 SDK，建议直接进入 Dev Client/Prebuild 路线。

---

## 4) 后端联调建议（前端不改页面）

- 前端所有“后端接入点”集中在 `src/repos/*.ts`。
- 后端同学可先按接口清单实现 API：`说明文档/后端接口清单.md`。
- 前端未来接入真实后端时，只替换 Repo 内部实现（Mock -> HTTP），页面层尽量不动。
