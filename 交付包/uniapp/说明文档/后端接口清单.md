# 后端接口对接清单（uni-app 版本，以 Repo 层为准）

目标：后端同学只要看这份文档 + `uniapp/src/services/repos/*` 里的 TypeScript 类型，就能实现接口并完成联调。

范围：uni-app 前端以 `uniapp/src/services/repos/*` 为“后端接入入口”，页面不会直接写真实网络请求；未来接入后端时只需要替换 Repo 内部实现即可。

---

## 0) 总体约定

### 0.1 返回结构（建议与前端 `Result<T>` 对齐）

前端通用结构见：`uniapp/src/services/types.ts`。

建议后端所有接口统一返回：

```ts
type ApiResult<T> =
  | { ok: true; data: T }
  | { ok: false; error: { code: string; message: string; detail?: string } };
```

分页结构建议统一为：

```ts
type PagedResult<T> = { items: T[]; page: number; pageSize: number; hasMore: boolean };
```

> 说明：前端页面目前主要使用 `page/pageSize/hasMore` 的分页风格；后端只要保证能映射到上述结构即可。

### 0.2 API 前缀

建议统一使用：`/api/v1/*`。

### 0.3 鉴权

- Header：`Authorization: Bearer <token>`
- 用户身份建议从 token 解析（避免业务接口重复传 `userId`）。

### 0.4 时间格式

建议使用 ISO 字符串（如 `2024-12-05T09:30:00.000Z`）或 `YYYY-MM-DD HH:mm`（当前 Mock 多为后者）。

### 0.5 文件上传（发帖图片/头像）

uni-app 端后续会走 `uni.uploadFile`。建议提供：

- `POST /api/v1/uploads`（multipart/form-data）→ `ApiResult<{ url: string }>`

## 1) 登录/认证（Auth）

对应：`uniapp/src/services/state/auth.ts`（当前为本地占位）。

- `POST /api/v1/auth/sms/send` → `ApiResult<{ requestId: string }>`
  - body：`{ phone: string }`
- `POST /api/v1/auth/sms/verify` → `ApiResult<{ token: string; user: { id: string; nickname: string; city?: string; vipLevel?: 'seed'|'growth'|'harvest' } }>`
  - body：`{ phone: string; code: string; requestId: string }`
- `POST /api/v1/auth/logout` → `ApiResult<{ ok: true }>`
- `GET /api/v1/me` → `ApiResult<{ id: string; nickname: string; city?: string; vipLevel?: 'seed'|'growth'|'harvest' }>`

## 4) 企业活动/预约/组团（Event / Booking / Group）

### 4.1 企业活动（Event）

对应：`uniapp/src/services/repos/event.ts`

- `GET /api/v1/companies/{companyId}/events?page=&pageSize=&date=` → `ApiResult<PagedResult<CompanyEvent>>`
- `GET /api/v1/companies/{companyId}/events/summary?startDate=&endDate=` → `ApiResult<Array<{ date: string; count: number }>>`

`CompanyEvent` 类型：见 `uniapp/src/services/repos/event.ts`。

### 4.2 预约（企业活动的 booking 模式）

对应：`uniapp/src/services/repos/event.ts` 的 `book(payload)`

- `POST /api/v1/bookings` → `ApiResult<{ bookingId: string }>`
  - body：`{ companyId: string; eventId: string; date: string; people: number; identity: string; note?: string }`

### 4.3 组团（企业活动的 group 模式）

对应：`uniapp/src/services/repos/event.ts` 的 `joinGroup(payload)`

- `POST /api/v1/groups/join` → `ApiResult<{ groupId: string }>`
  - body：`{ companyId: string; eventId: string }`

### 4.4 运营审核（预约列表/审核）

对应：`uniapp/src/services/repos/event.ts` 的 `listBookings/auditBooking`

- `GET /api/v1/ops/bookings?page=&pageSize=&companyId=` → `ApiResult<PagedResult<BookingRequest>>`
- `POST /api/v1/ops/bookings/{bookingId}/audit` → `ApiResult<{ ok: true }>`
  - body：`{ action: 'approve'|'reject'; note?: string }`

### 4.5 考察团（Group）

对应：`uniapp/src/services/repos/group.ts`

- `GET /api/v1/groups?page=&pageSize=&companyId=` → `ApiResult<PagedResult<Group>>`
- `GET /api/v1/groups/{groupId}` → `ApiResult<Group>`
- `POST /api/v1/groups/{groupId}/join` → `ApiResult<{ ok: true }>`
  - body：`{ headcount: number }`

### 4.6 参团成员/报名记录（Booking）

对应：`uniapp/src/services/repos/booking.ts`

- `GET /api/v1/groups/{groupId}/bookings?page=&pageSize=` → `ApiResult<PagedResult<Booking>>`
- `POST /api/v1/groups/{groupId}/bookings` → `ApiResult<{ bookingId: string }>`
  - body：`{ companyId: string; identity: string; headcount: number; contactName: string }`

## 5) 爱买买圈（Feed / Post / Draft / Comment / 深度互动 / 运营风控）

### 5.1 信息流与作者主页（Feed）

对应：`uniapp/src/services/repos/feed.ts`

- `GET /api/v1/feed?page=&pageSize=&tab=&sort=` → `ApiResult<PagedResult<Post>>`
  - `tab` 建议：`recommend | following | company | mine`
  - `sort` 建议：`latest | earliest | relevant`
- `GET /api/v1/posts/{postId}` → `ApiResult<Post>`
- `GET /api/v1/authors/{authorId}` → `ApiResult<AuthorProfile>`
- `GET /api/v1/authors/{authorId}/posts` → `ApiResult<Post[]>`
- `POST /api/v1/authors/{authorId}/follow` → `ApiResult<{ ok: true }>`

### 5.2 发帖（Post）

对应：`uniapp/src/services/repos/post.ts`

- `POST /api/v1/posts` → `ApiResult<{ id: string }>`
  - body：见 `PostRepo.create(payload)` 的字段（标题/正文/标签/模板/图片/配乐/可见性/评论开关/同步企业等）

### 5.3 草稿箱（Draft）

对应：`uniapp/src/services/repos/draft.ts`

- `GET /api/v1/post-drafts?page=&pageSize=` → `ApiResult<PagedResult<Draft>>`
- `GET /api/v1/post-drafts/{draftId}` → `ApiResult<Draft>`
- `POST /api/v1/post-drafts` → `ApiResult<{ id: string }>`
  - body：`DraftInput`（见 `uniapp/src/services/repos/draft.ts`）
- `DELETE /api/v1/post-drafts/{draftId}` → `ApiResult<{ ok: true }>`

### 5.4 评论（Comment）

对应：`uniapp/src/services/repos/comment.ts`

- `GET /api/v1/comments?page=&pageSize=&targetType=&targetId=` → `ApiResult<PagedResult<Comment>>`
  - `targetType`：`post | wish`
- `POST /api/v1/comments/{commentId}/like` → `ApiResult<{ liked: boolean; likes: number }>`
  - body：`{ liked: boolean }`（前端传当前状态，后端返回新状态与计数）

### 5.5 深度互动（专家提问/打赏/合作意向）

对应：`uniapp/src/services/repos/interaction.ts`

- `POST /api/v1/interactions/expert-questions` → `ApiResult<ExpertQuestionTicket>`
  - body：`{ postId: string; question: string; contact?: string }`
- `GET /api/v1/interactions/expert-questions/{ticketId}` → `ApiResult<ExpertQuestionTicket>`
- `POST /api/v1/interactions/expert-questions/{ticketId}/assign` → `ApiResult<ExpertQuestionTicket>`（运营/系统）
- `POST /api/v1/interactions/expert-questions/{ticketId}/reply` → `ApiResult<ExpertQuestionTicket>`
  - body：`{ reply?: string }`

- `POST /api/v1/interactions/tips` → `ApiResult<TipOrder>`
  - body：`{ postId: string; amount: number; method: 'wechat'|'alipay'; message?: string }`
- `POST /api/v1/interactions/tips/{orderId}/pay` → `ApiResult<TipOrder>`
- `POST /api/v1/interactions/tips/{orderId}/confirm` → `ApiResult<TipOrder>`

- `POST /api/v1/interactions/cooperation-intents` → `ApiResult<{ id: string }>`
  - body：`{ postId: string; companyName: string; contactName: string; contactPhone: string; message?: string }`

### 5.6 内容运营/风控（精华/榜单/举报/审核）

对应：`uniapp/src/services/repos/content-ops.ts`

- `GET /api/v1/ops/featured` → `ApiResult<Post[]>`

## 6) 心愿池（Wish）

  - body：`{ content: string; type: WishType }`

闭环占位：
- `POST /api/v1/wishes/{wishId}/accept` → `ApiResult<{ ok: true }>`（企业接单）
  - body：`{ companyId: string }`
