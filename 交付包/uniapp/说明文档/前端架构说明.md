# 前端架构说明（uni-app 版本，给后端/联调同学）

目标：让后端同学快速定位“哪些接口要做、字段契约在哪里、前端会怎么用”。

---

## 1) 分层与职责

### 1.1 页面层（`uniapp/src/pages`、`uniapp/src/pages-sub`）

- 只负责：路由、UI 组合、调用 Repo 获取数据、处理 Loading/Empty/Error。
- 页面不会直接写真实网络请求（当前阶段大多是 Mock）。

### 1.2 Repo 层（`uniapp/src/services/repos/*`）——后端接入入口

- 所有需要后端实现的能力都集中在这里（商品、订单、帖子、心愿、预约、风控等）。
- 当前实现大多是 Mock 数据/本地 Storage 占位。
- 后续接入真实后端时，只需要替换 Repo 内部实现（Mock -> HTTP），页面层尽量不改。

> 后端接口清单见：`uniapp/说明文档/后端接口清单.md`

### 1.3 通用返回结构（`uniapp/src/services/types.ts`）

- `Result<T>`：Repo 统一返回壳（成功/失败）。
- `PagedResult<T>`：分页结构（`page/pageSize/hasMore/items`）。

### 1.4 本地状态（`uniapp/src/services/state/*`）

- 保存纯前端交互态/本地态（如：购物车角标、登录态占位、消息本地种子数据）。
- 需要跨端/跨设备一致的数据（订单/消息/关注/内容等）以后端为准。

### 1.5 HTTP 占位（`uniapp/src/services/http.ts`）

- 统一封装 `uni.request`（baseURL/headers/错误映射）。
- 目前多数 Repo 还未接入该层；后续会在 Repo 中集中切换为真实请求。

---

## 2) 对后端同学最重要的入口

1) `uniapp/说明文档/后端接口清单.md`：要实现哪些接口、参数/返回是什么  
2) `uniapp/src/services/repos/*`：每个模块的 TypeScript 类型定义（字段契约）与前端用途注释  
3) `uniapp/src/services/types.ts`：统一 `Result/PagedResult` 结构
