# 环境变量与配置（uni-app 版本：前端/后端联调）

目标：明确“Mock/真实后端切换、API 地址、鉴权”等配置约定，减少联调摩擦。

---

## 1) 当前现状

- 当前 `uniapp/src/services/repos/*` 大多为 Mock/本地 Storage 占位，不依赖真实后端即可运行页面。
- 后端实现完成后，前端会将 Repo 的 Mock 替换为 HTTP 调用。

---

## 2) 建议的环境变量（Vite）

建议在 `uniapp/` 下使用 Vite 环境变量（`.env.*`），并在后续接入时由 `uniapp/src/services/http.ts` 读取。

- `VITE_API_BASE_URL`
  - 示例：`https://api.example.com`
  - 说明：后端 API 根地址（建议 HTTPS）
- `VITE_USE_MOCK`
  - 可选值：`true` / `false`
  - 说明：是否强制使用 Mock（联调时设为 `false`）
- `VITE_API_TIMEOUT_MS`
  - 示例：`12000`
  - 说明：HTTP 超时（毫秒）

---

## 3) 鉴权建议

统一使用 Header：

- `Authorization: Bearer <token>`

说明：
- 前端的登录态占位在 `uniapp/src/services/state/auth.ts`，后续会对接真实登录接口并写入 token。
- 后端建议通过 token 解析用户身份，不建议要求前端在业务接口里传 `userId`。
