# 前端架构说明（给后端/新同学快速上手）

## 1) 分层与职责

### 1.1 页面层（`app/`）
- 只负责：路由、UI 组合、调用 Repo 获取数据、处理 Loading/Empty/Error。
- 不直接请求网络（不写 `fetch()`）。

### 1.2 Repo 层（`src/repos/`）
- 统一的数据访问入口（后端接入点）。
- 目前以 Mock 数据实现；后续接入后端时替换为 HTTP 调用即可。
- 统一返回 `Result<T>`，错误使用 `AppError`。

接口清单：`说明文档/后端接口清单.md`

### 1.3 Types（`src/types/`）
- `domain/*`：业务域模型（前后端字段契约第一版）。
- `Result<T>`：统一返回壳（成功/失败）。
- `AppError`：统一错误码/展示文案。

### 1.4 Store（`src/store/`）
- zustand：保存纯前端交互态/本地态（如购物车角标、数量修改）。
- 需要跨端同步的数据（订单/消息/关注等）以 Repo + 后端为准。

### 1.5 Theme / Components
- `src/theme`：自然绿主导 + 科技蓝点缀的主题系统。
- `src/components`：基础组件、业务卡片、反馈态、弹层等复用组件。

---

## 2) 错误处理约定（Result/AppError）

- Repo 方法返回 `Result<T>`：
  - `ok: true` → 页面展示数据
  - `ok: false` → 页面展示 ErrorState/Toast（可带重试）
- 错误码：`NETWORK | NOT_FOUND | FORBIDDEN | INVALID | UNKNOWN`

---

## 3) 对后端同学最重要的入口

1) `说明文档/后端接口清单.md`：要实现哪些接口
2) `src/types/domain/*`：字段/枚举/结构契约
3) `src/repos/*.ts`：每个接口在前端里的用途与调用位置

---

## 4) 中文注释规则（本项目约定）

- 公共组件 + Repo / Store / 复杂业务逻辑：需要中文注释（解释用途与关键决策）
- 普通页面组件与纯 UI 组合：不强制逐函数注释
