> 版本：1.0 | 创建时间：2026-02-18
> 技术栈：React Native 0.81 + Expo 54 + expo-router 6 + react-native-reanimated

## 1. 设计理念

### 1.1 核心概念：「脉」

「脉」是整个 App 的设计灵魂 —— 数字脉搏连接大地与智能。

- **大地之脉**：农业的根基，绿色、有机、生长的力量
- **科技之脉**：AI 的智能，光流、粒子、呼吸的节奏
- **人心之脉**：用户的需求，温暖、可信、自然的交互

这三条脉络交织形成独特的视觉语言：**有机生物形态（Organic Biophilic）+ AI 光效（AI Luminance）**。App 的每一处设计都应让用户感受到——这不是一个普通的电商 App，而是一个有生命感的 AI 农业伙伴。

### 1.2 设计原则

| 原则 | 含义 | 体现 |
|------|------|------|
| **AI 原生** | AI 不是附加功能，而是核心体验 | 首页即 AI 入口，全局 AI 伴侣 |
| **有机科技** | 自然形态 + 数字光效融合 | 圆角有机形状 + 粒子/渐变/辉光 |
| **温暖可信** | 农产品需要信任感 | 大字体、清晰层次、溯源可视化 |
| **呼吸感** | UI 有生命节奏 | 微动效、脉动、渐变流动 |
| **双模式一致** | 浅/深色模式同等精致 | 完整的双色彩系统 |

### 1.3 与传统电商的差异

| 维度 | 传统电商 | 爱买买 |
|------|---------|------|
| 首页 | 商品瀑布流/Banner 轮播 | AI 语音入口 + 对话式发现 |
| 搜索 | 文字搜索框 | 语音优先 + AI 语义理解 |
| 商品推荐 | 算法推荐列表 | AI 对话推荐 + 理由解释 |
| 商品详情 | 图文堆砌 | AI 品质评分 + 溯源可视化 |
| 导航 | 分类 Tab 驱动 | AI 引导 + 传统浏览并存 |
| 视觉风格 | 扁平/白底/卡片式 | 有机形态 + 光效粒子 + 渐变 |

## 3. AI 设计语言

### 3.1 AI 光球（The Orb）

AI 光球是「脉脉」的视觉化身，贯穿整个 App。

#### 光球形态

```
外层：模糊光晕（blur 40px, aiGlow 色, 20% 透明度）
中层：渐变圆形（aiOrbGradient）
内层：亮白色核心点（4px, 80% 透明度）
环绕：2-3 圈半透明同心环（aiAccent, 5%-15% 透明度）
粒子：8-12 个微小光点在光球周围缓慢漂浮
```

#### 光球状态动效

| 状态 | 视觉 | 动效 |
|------|------|------|
| **闲置 (Idle)** | 缓慢呼吸脉动 | 缩放 0.95↔1.05，周期 3s，粒子缓慢漂浮 |
| **聆听 (Listening)** | 光球变大，光环扩散 | 缩放 1.0→1.15，脉冲环向外扩散，粒子向内聚集 |
| **思考 (Thinking)** | 快速旋转微光 | 内部渐变旋转，粒子加速环绕，颜色微微偏移 |
| **回应 (Responding)** | 柔和绽放 | 一次向外脉冲，粒子释放，然后回到 Idle |
| **错误 (Error)** | 抖动 + 变暗 | 快速左右抖动 2 次，光球暗淡 0.5s 后恢复 |

#### 首页大光球（主入口）

尺寸：160×160pt（加光晕约 240×240）

```
      ╭─ ─ ─ ─ ─ ─ ─╮
    ╱   · ·  ·  · ·    ╲         ← 漂浮粒子层
   │    ╭─────────╮      │
   │   ╱ ░░░░░░░░░ ╲     │       ← 模糊光晕层
   │  │  ╭───────╮  │    │
   │  │  │  AI   │  │    │       ← "AI" 超大字（36pt+，粗体，居中）
   │  │  │ 买买  │  │    │       ← "买买" 较小字（16pt，居 AI 下方）
   │  │  ╰───────╯  │    │
   │   ╲ ░░░░░░░░░ ╱     │
   │    ╰─────────╯      │
    ╲   · ·  ·  · ·    ╱
      ╰─ ─ ─ ─ ─ ─ ─╯
```

球体内文字排版：
- **"AI"**：`displayMd`（36pt），`fontWeight: 800`，白色，居中，占据球体上半视觉重心
- **"买买"**：`bodyStrong`（16pt），白色，紧跟 AI 下方（间距 2px），形成上大下小的层次感
- 整体效果：AI 二字占据球体约 60% 视觉面积，强烈凸显 AI 身份

长按光球触发语音输入，松开发送。录音状态下文字变为「正在听...」+ 麦克风图标。

#### 全局浮动小光球

尺寸：48×48pt（加光晕约 64×64）
位置：右下角（距底部 Tab 栏上方 16px，距右边 16px）
行为：

- 轻触 → 展开 AI 快捷菜单（3-4 个上下文相关选项）
- 长按 → 直接进入语音输入
- 左滑 → 隐藏到屏幕边缘（只露出半圆弧光）
- 在首页 Tab 时自动隐藏（避免与大光球重复）
- 在特定交互中（如支付、表单填写）自动缩小为微型光点

### 3.2 AI 上下文菜单

浮动小光球展开后显示的快捷选项（根据当前页面动态变化）：

```
                    ╭────────────╮
                    │  问问脉脉   │ ← 进入 AI 聊天
                    ├────────────┤
                    │ {上下文选项} │ ← 根据页面变化
                    ├────────────┤
                    │ {上下文选项} │
                    ╰────────────╯
                          ◉       ← 小光球
```

| 当前页面 | 上下文选项 |
|---------|-----------|
| 商品详情 | 「这个值得买吗？」「有没有类似的？」 |
| 购物车 | 「帮我凑个满减」「推荐搭配商品」 |
| 订单列表 | 「我的快递到哪了？」「申请售后」 |
| 企业详情 | 「这家靠谱吗？」「看看评价」 |
| 钱包/VIP | 「怎么提现？」「升级 VIP 有什么好处？」 |
| 其他 | 「帮我找商品」「今天吃什么？」 |

### 3.3 AI 微交互元素

这些元素散布在各页面，统一传达 AI 存在感：

#### AI 标签 (AI Badge)

```
 ╭──────────╮
 │ ✦ AI推荐  │    ← 渐变边框(aiGradient)，内部微光扫过动效
 ╰──────────╯
```

变体：`AI推荐` / `AI评分` / `AI溯源` / `AI分析` / `脉脉精选`

#### AI 分割线 (AI Divider)

普通分割线用静态细线，AI 区域的分割线用渐变色 + 中间微光点移动效果：

```
────── · ──────    普通分割线
━━━━◉━━━━━━━━━    AI 分割线（渐变色 + 光点从左到右缓慢移动）
```

#### AI 加载态 (AI Loading)

替代传统 Spinner，使用光球脉动 + 文字：

```
     ◉         ← 小光球脉动
  脉脉正在思考...  ← AI 风格加载文字（字符逐个显示）
```

#### AI 卡片光边 (AI Card Glow)

AI 生成/推荐的内容卡片，左侧有 2px 渐变光边：

```
 ┃╭──────────────────╮
 ┃│ AI 推荐的商品卡片  │    ← 左侧 2px 渐变条(aiGradient)
 ┃│  ...              │
 ┃╰──────────────────╯
```

### 3.4 脉动粒子系统

粒子效果是 AI 存在感的核心视觉，但需克制使用以保持性能：

| 场景 | 粒子数 | 行为 | 性能消耗 |
|------|--------|------|---------|
| 首页大光球 | 8-12 | 围绕光球缓慢漂浮 | 中 |
| 语音输入中 | 16-20 | 向光球聚集 | 中 |
| 浮动小光球 | 3-4 | 缓慢环绕 | 低 |
| 页面过渡 | 6-8 | 从中心向外扩散后消失 | 低（一次性） |
| 背景粒子（首页） | 15-20 | 极慢漂浮，低透明度 | 低 |

所有粒子使用 `react-native-reanimated` 的 `useSharedValue` + `useAnimatedStyle` 实现，运行在 UI 线程。

**减弱动效模式**：检测系统 `prefers-reduced-motion`，关闭粒子和连续动画，保留基本过渡。

#### 设计细节

- **问候语**：根据时段变化（早上好/下午好/晚上好），使用 `displaySm` 字号
- **AI 引导语**：每次打开随机显示不同的 AI 引导语，如：
  - 「今天想吃点什么？」
  - 「有什么可以帮你的？」
  - 「看看今天有什么新鲜的？」
  - 「说出你想要的，我帮你找」
- **搜索框**：胶囊形，背景 `surface`，点击后跳转独立搜索页
- **大光球**：页面核心，占据视觉中心约 40% 的区域
  - 球内文字 **"AI"** 超大（36pt，fontWeight 800），**"买买"** 小字（16pt）紧贴其下
  - 整体上大下小排列，"AI" 占据球体视觉主体，强烈凸显 AI 身份
- **快捷指令气泡**：胶囊形标签，背景 `surface`，边框 `border`，文字 `textPrimary`
  - 内容由后端返回（基于用户历史和季节推荐）
  - 点击后直接将指令发给 AI，跳转 AI 聊天页
- **最近对话**：展示最近 2-3 条 AI 对话摘要
  - 左侧小光球图标（aiAccent 色）
  - 点击恢复该对话上下文
  - 无对话时显示「和脉脉聊聊，发现更多好物」

### 5.1b 每日抽奖页 `/lottery` ✅

从首页右下角浮动按钮（FAB）进入。以可视化转盘为核心的游戏化抽奖体验。

```
┌─────────────────────────────┐
│  ← 每日抽奖                  │  AppHeader
├─────────────────────────────┤
│                              │
│  ┌──────────┐                │  剩余次数胶囊
│  │  剩余1次  │                │  (gold.light 背景 pill)
│  └──────────┘                │
│                              │
│         ▽ 指针               │  WheelPointer（固定不旋转）
│    ┌──────────┐              │
│    │  转盘区域  │              │  SpinWheel（280×280 SVG）
│    │  4扇区等分 │              │  扇区颜色按奖品类型映射
│    └──────────┘              │
│     ✨ Confetti              │  Confetti（中奖粒子爆发层）
│                              │
│     ┌──────────┐             │
│     │   抽奖    │             │  DrawButton（72px 金色圆形）
│     └──────────┘             │  空闲脉冲 1.0↔1.05
│                              │
│  ┌─ 奖品列表 ──────── ▼ ─┐   │  可折叠手风琴
│  │ ● 5元红包  ● 免邮券 ... │   │
│  └──────────────────────┘   │
│                              │
│  〔结果底部弹窗〕              │  AppBottomSheet
└─────────────────────────────┘
```

#### 核心交互

**状态机**：`idle → spinning → decelerating → revealing → result_shown → idle`

| 阶段 | 时长 | 动画 |
|------|------|------|
| spinning | 2000ms | 5圈匀速旋转（Easing.linear），同时发起 API |
| decelerating | 2500ms | 减速到目标扇区（Easing.bezier(0.15,0.85,0.25,1)），至少+3圈 |
| revealing | 700ms | 中奖：Confetti 25粒子爆发 + 转盘闪烁3次；未中奖：轻透明度呼吸 |
| result_shown | — | AppBottomSheet 弹出，中奖用 AiTypingEffect 逐字揭晓奖品名 |

**结果弹窗**：
- 中奖：🎁 大图标 + AiTypingEffect 奖品名 + "奖品已自动发放" + 金色"太棒了!"按钮
- 未中奖：😊 图标 + "谢谢参与，明天再来" + 普通"好的"按钮

**扇区颜色映射**：RED_PACK → `gold.primary`，COUPON → `brand.primaryLight`，PRODUCT → `danger`，NONE → `bgSecondary`

#### 边界情况
- API < 2s 返回：存 ref 等待快速旋转阶段完成再减速
- API > 10s：超时保护，强制停转显示失败
- 奖品 < 2 个：显示 EmptyState 替代转盘

## 6. 动效规范

### 6.1 时长标准

| 类型 | 时长 | 缓动函数 | 用途 |
|------|------|---------|------|
| 微交互 | 150ms | `easeOut` | 按钮 press、切换 |
| 标准过渡 | 250ms | `easeInOut` | 页面切换、展开/折叠 |
| 强调动效 | 350ms | `spring(damping:15)` | AI 光球变化、卡片入场 |
| 连续动画 | 3000ms | `linear` 循环 | 光球呼吸、粒子漂浮 |

### 6.2 入场动画

列表元素进入视野时的动画（stagger）：

```
每个元素: opacity 0→1 + translateY 20→0
间隔: 50ms
时长: 250ms
缓动: easeOut
```

### 6.3 手势反馈

| 手势 | 反馈 |
|------|------|
| 轻触按钮 | scale 0.97 + opacity 0.8, 150ms |
| 轻触卡片 | scale 0.98 + shadow 减弱, 150ms |
| 长按光球 | scale 1.15 + 光环扩散 + Haptic (medium) |
| 下拉刷新 | AI 光球出现在顶部 + 旋转 |
| 左滑删除 | 红色背景滑出 + 垃圾桶图标 |

### 6.4 减弱动效模式

当系统开启 `Reduce Motion` 时：
- 关闭所有粒子动画
- 光球呼吸脉动替换为静态发光
- 列表入场动画替换为简单 opacity 过渡
- 页面过渡替换为简单 crossfade
- 保留功能性动画（加载 spinner、进度条）

## 9. 技术实现备注

### 9.1 主题切换

使用 React Context + `useColorScheme()` 检测系统主题：

```
ThemeProvider
├── useColorScheme() → 'light' | 'dark'
├── lightTheme / darkTheme 完整色彩对象
├── useTheme() hook → 返回当前主题
└── 所有组件通过 useTheme() 获取颜色
```

### 9.2 动效技术选型

| 动效类型 | 技术 |
|---------|------|
| 光球脉动/粒子 | `react-native-reanimated` SharedValue + AnimatedStyle |
| 手势（长按/拖拽）| `react-native-gesture-handler` |
| 列表入场动画 | `reanimated` entering/exiting |
| 页面过渡 | expo-router 自定义 transition |
| Shimmer 效果 | `react-native-reanimated` + LinearGradient |
| 毛玻璃 | `expo-blur` BlurView |

### 9.3 性能守则

- 粒子动画全部运行在 UI 线程（worklet）
- 长列表使用 `FlashList` 替代 `FlatList`
- 图片使用 `expo-image`（内置缓存 + 渐进加载）
- 首页粒子在 App 进入后台时暂停
- 深色模式预计算所有颜色值，避免运行时计算
