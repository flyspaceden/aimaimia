# 电商展示商城设计方案

**日期：** 2026-04-03  
**目的：** 在现有 website/ 官网内嵌入一套完整的电商展示商城，供 App Store 审核员验证平台电商属性。  
**约束：** 纯前端 mock，不连后端，不需要真实支付/登录逻辑。

---

## 1. 项目背景

爱买买 App 需要通过应用商店审核，审核员要求平台方提供可访问的网站，证明其是真实运营的电商平台。现有 website/ 是企业官网（暗色科技风），需在其基础上新增一套可独立访问的电商展示模块。

---

## 2. 架构决策

### 方案：完全独立的商城布局

商城路由（`/shop/*`）使用**独立的 ShopLayout**，不复用现有官网 Navbar/Footer。官网 App.tsx 检测路由前缀，对 `/shop/*` 隐藏原有 Navbar/Footer，改由 ShopLayout 内部的 ShopNavbar 和 ShopFooter 渲染。

**优势：**
- 视觉上是完整的电商平台，审核说服力最强
- 响应式设计不受官网暗色主题约束
- 商城内导航（搜索、购物车、分类）独立管理

**入口：** 官网 Navbar 新增"进入商城"按钮，点击后 `<Link to="/shop">` 跳转。

---

## 3. 路由结构

```
/shop                     → ShopHome       商城首页
/shop/category/:id        → ShopCategory   分类商品列表
/shop/product/:id         → ShopProduct    商品详情
/shop/cart                → ShopCart       购物车
/shop/checkout            → ShopCheckout  结账付款页
/shop/user                → ShopUser       个人中心（模拟登录态）
```

所有 `/shop/*` 路由均使用 `ShopLayout` 包裹，官网 Navbar/Footer 在这些路由下隐藏。

---

## 4. 视觉设计规范

### 色彩
- **主色：** #2E7D32（品牌绿）
- **背景：** #ffffff / #f5f5f5（浅色电商风）
- **价格：** #e53935（红色，电商惯例）
- **CTA 按钮：** #FF6F00（橙色，立即购买）
- **分类标签激活：** #2E7D32 + 2px 下划线

### 响应式断点
| 屏幕 | 商品列数 |
|------|---------|
| 手机 < 640px | 2列 |
| 平板 640px–1024px | 3列 |
| 桌面 > 1024px | 4列 |

### Tailwind 实现
使用 `grid-cols-2 sm:grid-cols-3 lg:grid-cols-4` 实现响应式商品网格。

## 5. 页面设计

### 5.1 商城首页（ShopHome）

**结构（从上到下）：**
1. **ShopNavbar**：Logo + 搜索框 + 城市选择 + 我的订单 + 购物车（徽章）+ 用户头像
2. **分类标签栏**：全部 / 海鲜水产 / 鱼类 / 新鲜水果 / 时令蔬菜 / 肉禽蛋 / 农家特产 / 冷冻食品（横向滚动，移动端可滑动）
3. **Hero Banner**：全宽渐变背景（深绿→品牌绿→青绿），标语 + 副标题 + CTA 按钮
4. **限时秒杀条**：红色标签 + "今日特惠" + 倒计时（静态展示）+ "查看全部"
5. **商品分区**（每区含标题 + "查看全部" + 响应式商品网格）：
   - 🦀 爆款海鲜（8件）
   - 🍊 当季水果（4件）
   - 🥬 时令蔬菜（4件）
6. **ShopFooter**：深绿背景 + 粤ICP备案号 + 食品经营许可证 + 客服热线

### 5.2 分类页（ShopCategory）

- 顶部面包屑：首页 › 海鲜水产
- 左侧子分类导航（桌面）/ 顶部横向滑动标签（移动端）：螃蟹 / 虾类 / 鱼类 / 贝类 / 其他
- 主区域：排序工具栏（综合/销量/价格↑↓）+ 响应式商品网格
- 商品卡：图片 + 名称 + 产地标签 + 价格 + 加入购物车按钮

### 5.3 商品详情页（ShopProduct）

**桌面：左图右信息两栏布局；移动端：上图下信息单列**

- **图片区**：大图（emoji + 渐变背景）+ 4个缩略图（可点击切换，mock）
- **信息区**：
  - 商品标题（含产地）
  - 价格区：现价（红） + 原价（删除线） + 折扣标签
  - 标签行：顺丰冷链 / 产地直发 / 活蟹保鲜等
  - 规格选择（SKU）：多个尺寸/重量选项，点击高亮
  - 数量控制：− / 数字 / + + 库存显示
  - 操作按钮：加入购物车（绿色描边）/ 立即购买（橙色实心）
- **详情区**：商品属性表格 + 文字介绍段落
- **面包屑**导航

### 5.4 购物车页（ShopCart）

**桌面：左列表右摘要两栏；移动端：单列堆叠**

- **商品列表**：全选复选框 + 每项（图片/名称/规格/价格/数量±/删除）
- **订单摘要**：
  - 优惠券入口（展示入口，mock 无可用券）
  - 商品总价 / 运费（免运费）/ 优惠
  - 合计（红色大字）
  - 结算按钮（橙色）
  - 安全支付说明
  - 猜你喜欢（2个商品推荐）

### 5.5 结账付款页（ShopCheckout）

**入口：** 购物车页"结算"按钮 → `/shop/checkout`

**桌面：左侧订单信息 + 右侧支付摘要两栏；移动端：单列堆叠**

- **左侧 - 收货信息**：
  - 收货地址区（mock 地址：广州市天河区某小区，含姓名+手机号）
  - 商品清单（缩略图 + 名称 + 规格 + 数量 + 小计）
  - 配送方式选择：顺丰冷链（¥0）/ 次日达（¥12）（radio 选择，mock）
  - 买家备注输入框（textarea，不实际提交）

- **右侧 - 支付摘要**：
  - 商品合计 / 运费 / 优惠
  - 实付金额（红色大字）
  - 支付方式选择：微信支付 / 支付宝 / 银行卡（radio 图标选择）
  - **"确认支付"按钮**（橙色大按钮）→ 点击后弹出"支付成功"提示弹窗（mock），弹窗含订单号 + "查看订单"按钮 → 跳转个人中心
  - 安全加密说明（🔒 SSL 加密保护）

### 5.6 个人中心（ShopUser）

- 顶部用户信息区：头像 + 姓名"李**" + 会员等级标签
- 数据统计：待付款 / 待发货 / 待收货 / 已完成（mock 数字）
- 功能菜单列表：我的订单 / 收货地址 / 优惠券 / 浏览历史 / 帮助中心 / 退出登录

## 7. 组件结构

```
website/src/
├── pages/shop/
│   ├── ShopHome.tsx        # 商城首页
│   ├── ShopCategory.tsx    # 分类列表页
│   ├── ShopProduct.tsx     # 商品详情页
│   ├── ShopCart.tsx        # 购物车页
│   ├── ShopCheckout.tsx    # 结账付款页
│   └── ShopUser.tsx        # 个人中心
├── components/shop/
│   ├── ShopLayout.tsx      # 商城外层布局（含 Navbar + Footer）
│   ├── ShopNavbar.tsx      # 商城专属导航栏（响应式）
│   ├── ShopFooter.tsx      # 商城底部（含备案信息）
│   └── ProductCard.tsx     # 商品卡片组件（响应式）
└── data/
    └── shopMockData.ts     # 所有 mock 商品/分类数据
```

## 8. 对现有文件的改动

### `website/src/App.tsx`
- 新增 `/shop/*` lazy 路由，指向 `ShopHome/ShopCategory/ShopProduct/ShopCart/ShopUser`
- `isLandingPage` 逻辑扩展：加入 `location.pathname.startsWith('/shop')` → 隐藏官网 Navbar/Footer

### `website/src/components/layout/Navbar.tsx`
- 在"下载 App"按钮左侧新增**"进入商城"**按钮（`<Link to="/shop">`）
- 桌面端显示，移动端菜单中也加入该入口
- 样式：绿色描边 / 绿色文字，与现有暗色导航协调

---

## 9. 购物车状态管理

使用 React `useState` 本地状态（不引入 Zustand/Context，范围最小化）：
- `ShopCart.tsx` 维护 `cartItems` 数组（mock 初始化 3 件商品）
- 支持：增减数量、勾选/取消、删除（纯 UI 操作，不持久化）
- 购物车徽章数量从 `ShopNavbar` 接收 props 或使用简单 Context
