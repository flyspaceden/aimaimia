# VIP/普通配置利润安全实时预检设计

## 目标

让管理员在 VIP 系统配置或普通系统配置页面修改七项利润比例时，在不保存的情况下自动看到该候选配置与当前团长 V3 配置、当前在售 SKU 叠加后的利润安全结果；后端保存时的原子硬校验保持不变。

## 背景与边界

当前三页的后端保存均使用 `ProfitSafetyService` 做 Serializable + advisory lock 的候选快照校验，任何不安全的保存都会整体拒绝。团长配置页已经在保存前调用 `POST /admin/config/profit-safety-preview` 并展示候选结果。VIP、普通配置页只展示已保存配置的安全状态，虽然七项比例合计会实时显示，但管理员无法在修改过程中看到候选配置的限制 SKU 和利润缺口。

本次只补 VIP 与普通配置页的候选预检交互：不修改分润公式、订单快照、保存接口、权限、数据库结构或团长配置页。

## 交互设计

### 触发与节流

1. 表单加载完成前不发预检请求。
2. 管理员修改任一 VIP/普通配置字段后，页面维持既有未保存变更状态。
3. 仅当七项比例合计为 100%、存在实际配置变化且无字段校验错误时，等待 500ms 后调用预检接口。
4. 预检请求使用当前表单中全部发生变化的 `RuleConfig` 键值，而不是只传单个刚编辑的比例。
5. 连续输入期间取消旧计时器；过期响应不得覆盖最新一次表单状态。

### 展示

1. 未改动时继续显示服务器已保存配置的 `ProfitSafetyStatus`。
2. 候选预检中显示“正在校验未保存参数”的信息状态。
3. 候选安全时展示“未保存参数通过校验”，包括已检查 SKU 数、平台所需留存、团长最高利润奖励。
4. 候选不安全时展示红色结果，包含失败组合、限制 SKU、利润缺口、后端错误原因，并提供跳转团长配置的既有入口。
5. 比例合计不是 100% 时，不发送候选预检；保留页面现有的比例合计错误提示，并显示“需先使七项比例合计为 100% 才能校验利润安全”。
6. 预检网络失败时显示警告，不把失败当成安全，也不影响管理员继续编辑；保存仍由后端硬拦截兜底。

### 保存关系

1. 候选预检只读，不写配置、不写版本历史、不写审计日志。
2. 点击保存时继续走现有 `PUT /admin/config/batch` 原子批量写入。
3. 保存接口仍在同一安全锁和 Serializable 事务内重新计算完整候选快照；浏览器预检成功不能替代最终保存校验。
4. 保存成功后清除候选状态并刷新服务器已保存配置的安全摘要。
5. 保存失败后保留当前未保存表单值和最新候选摘要，管理员可据此继续调整。

## 技术设计

新增前端 hook `useConfigProfitSafetyPreview`，负责从 `CONFIG_SCHEMA` 与当前 `configs` 构造 `updates`：

```ts
type PreviewState = {
  mode: 'saved' | 'checking' | 'candidate' | 'invalid-ratio' | 'error';
  summary?: ProfitSafetySummary;
  error?: Error;
};

useConfigProfitSafetyPreview({
  configs,
  values: formValues,
  ratioKeys: RATIO_KEYS,
  enabled: configs.length > 0,
  delayMs: 500,
});
```

hook 复用现有 `previewProfitSafety({ updates })` API。每次请求携带递增序号，只有与当前序号一致的响应可以更新状态。组件层继续复用 `ProfitSafetyStatus`，只为其增加可选的“候选/校验中/比例无效”展示信息，避免 VIP、普通、团长页各自复制结果卡片。

## 验收标准

1. 在 VIP 配置页把直推比例改高且七项仍为 100% 后，500ms 内可看到未保存候选的限制 SKU 和利润缺口；不点击保存不会写任何配置。
2. 普通配置页具有同样行为。
3. 七项比例不为 100% 时不调用预检接口，并保留既有保存拦截。
4. 快速连续编辑时，旧预检响应不会覆盖最新结果。
5. 预检失败显示警告；后端保存硬拦截仍然存在。
6. 保存成功后页面回到服务器已保存摘要；保存失败后保留候选结果和表单值。
7. 管理后台 TypeScript、生产构建、现有团长静态测试及新增预检测试通过。

## 非目标

- 不把团长比例放入 VIP 七项比例或普通七项比例。
- 不在订单结算时做动态缩放。
- 不修改现有 `ProfitSafetyService` 的安全公式。
- 不改动历史 V2 订单、VIP 树、普通树、团长绑定关系或支付链路。
