# 团长一层直推经营激励设计方案

## 1. 决策与目标

预包装海鲜团长计划改为一层直推经营模式。团长只能因自己直接推广的真实消费者购买合规商品而获得收益；任何团长都不能因另一名团长或该团长客户的成交获得订单佣金、月度 GMV 或奖金。

这项改造不修改 VIP 树、普通分润树、Reward 消费积分或 Coupon 平台红包。团长码仍是独立的 `/c/{code}` 路径，但只建立买家与直接团长的一条关系。

## 2. 一层经营规则

### 2.1 绑定与关系

1. 买家使用团长码后，只写入 `directCaptainUserId`。
2. 即使该直接团长本身已绑定其他团长，也不得向上追溯或生成任何间接团长。
3. 买家后续独立申请并获批成为团长，不成为原团长的下级团长；其后续客户也不与原团长产生经营或收益关系。
4. 一个买家在一个团长项目内仍只能绑定一名直接团长，重复绑定同一团长幂等，换绑继续拒绝。

### 2.2 订单收益

1. 合格商品订单只创建一笔 `DIRECT_ORDER` 冻结流水，受原有收货、售后期、退款冲回和幂等保护约束。
2. 默认直接成交佣金率为 11%。这是原 9% 直接佣金与 2% 间接佣金的预算合并，不增加总激励上限。
3. 不再创建 `INDIRECT_ORDER` 流水，不再向任何上级团长归因，也不再保存新的间接佣金费率。

### 2.3 月度经营奖励

月度指标只统计该团长直接客户的有效净商品 GMV。原“团队 GMV”“团队成员”“新人培育”和“团队池”全部改为直接经营口径：

| 原口径 | 新口径 |
| --- | --- |
| 个人 GMV / 团队 GMV | 直接客户有效 GMV |
| 团队有效成员 | 有效直接客户 |
| 新有效成员 | 当月新增有效直接客户 |
| 团队分红池 40%/60% | 经营绩效奖，100% 发给本团长 |

默认资格为有效直接客户不少于 12 人、当月直接客户有效净 GMV 不低于 8,000 元、当月新增有效直接客户不少于 1 人，并继续受退款率和运营任务风控约束。基础、增长、卓越档门槛维持 25,000 / 70,000 / 140,000 元，全部基于直接客户有效净 GMV。

满配激励率保持 15.5%：直接成交 11% + 管理津贴 2.2% + 增长奖 0.7% + 有效成交辅导奖 0.6% + 经营绩效奖 1%。所有比例仍由后台配置并受 15.5% 上限校验。

## 3. 历史数据与切换

已存在的二级关系、订单归因和 `INDIRECT_ORDER` 账本不删除，作为历史审计资料保留；其已经冻结、可用、已支付或冲回的状态机仍可按既有规则完成，避免篡改既有财务凭证。

新版本不再读取或写入二级关系来产生新的订单收益或月度奖励。管理后台对历史二级账本标注“历史二级佣金”，不再将它呈现为可配置或可继续运营的激励项目。

## 4. 数据与接口边界

由于财务台账需要保留，数据库中的历史间接字段和枚举暂不物理删除。它们只服务于历史查询与冻结账本收尾；当前 API、配置 DTO、买家端类型和管理后台页面不再暴露可运营的二级字段。

现有月度结算金额字段继续保存“经营绩效奖”金额以避免破坏历史结算记录；业务语义、后台标签和新的配置字段均切换为一层直接经营口径。

## 5. 风控与验收

1. 任意绑定链路最多只产生一个直接团长。
2. 任意新订单最多创建一笔团长订单佣金流水。
3. 间接团长停用、启用或关系变化不影响新订单佣金。
4. 月度 GMV、有效客户、新客户及绩效奖均只来自直接归因订单和直接绑定客户。
5. 历史 `INDIRECT_ORDER` 流水可释放、作废、冲回和查询，但任何新支付订单不会创建该类型。
6. 配置总激励率固定按 11% + 2.2% + 0.7% + 0.6% + 1% 计算，超过 15.5% 拒绝保存。
7. 所有账户、账本、结算和配置写入继续使用 `Serializable` 事务、幂等键和现有结算状态机。

这是一项产品与工程合规控制，不替代律师对最终合同、培训话术、营销页面和实际运营的专项法律意见。
