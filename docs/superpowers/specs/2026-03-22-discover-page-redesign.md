# 发现页 UIUX 重设计方案

## 概述

优化买家 App 发现页（museum.tsx），将商品和企业浏览通过顶部 Tab 切换清晰分离，同时通过交叉引流保持两者关联。AI 智能感体现在搜索和推荐层面。

**范围**：仅发现页，首页不动。

## 设计决策记录

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 商品/企业布局 | 顶部 Tab 切换（A方案） | 切换成本最低，两个维度各自独立 |
| 商品/企业割裂问题 | 交叉引流 | 商品卡片显示企业来源，企业卡片显示热门商品 |
| AI 智能感重点 | 搜索 + 推荐 | 信息增强（溯源评分等）后续再做 |
| 地图模式 | 全屏地图 + 浮动卡片（C方案） | 地图空间最大化，交互沉浸感强 |
| AI 推荐区布局 | 横向滑动等高卡片 | 精选少量，横滑浏览，与下方瀑布流形成节奏对比 |
| 热门商品布局 | 瀑布流不等高 | 参差错落更有浏览感，纵向无限滚动 |

## 页面整体结构

```
┌─────────────────────────┐
│  标题行：发现    🗺  🛒  │  ← 固定顶部（地图切换 + 购物车）
├─────────────────────────┤
│  🔍 搜索商品、品类、企业 │  ← 固定顶部，简洁搜索框
├─────────────────────────┤
│    商品    |    企业     │  ← 固定 Tab，吸顶
├─────────────────────────┤
│  分类/筛选标签横滑       │  ← 跟随 Tab 变化
├─────────────────────────┤
│  内容区（按 Tab 切换）   │
│                         │
│        ∞ 无限滚动        │
├─────────────────────────┤
│   首页   |  发现  | 我的  │  ← 底部导航栏
└─────────────────────────┘
```

关键要点：
- **搜索框和 Tab 吸顶**：滚动时始终可见
- **搜索框统一**：不分 Tab，搜索时 AI 自动判断意图（商品/企业），利用已有语义意图识别能力
- **搜索框样式**：简洁普通搜索框，不需要 AI 标识和麦克风图标
- **分类标签跟随 Tab 切换**：商品 Tab 显示品类，企业 Tab 显示筛选维度

## 商品 Tab

### 分类标签横滑
- 一级品类标签：全部 / 🍎 水果 / 🥬 蔬菜 / 🍵 茶叶 / 🌾 粮油 / ...
- 选中态：绿色填充背景 + 绿色边框
- 未选中态：白色背景 + 灰色边框
- 选中分类后筛选下方商品列表

### AI 推荐区（脉脉精选）
- **布局**：横向滑动，等高卡片
- **标题行**：`✦ 脉脉精选` AI 渐变标签 + "为你推荐" 副文案
- **卡片内容**：
  - 商品图片（等高）
  - AI推荐标签（左上角渐变角标）
  - 商品名称
  - 推荐理由（一行小字）
  - 价格 + 月销量
  - 企业来源标签（底部分隔线下方，`🌿 绿源农场 >`，可点击跳转企业页）
- **数据来源**：后端 AI 推荐接口，基于用户浏览偏好和购买历史
- **卡片数量**：4-8 张，最后一张半露出提示可继续滑动

### AI 推荐与热门商品分隔
- 灰色分隔线（6px 高，`#f5f5f5` 背景），视觉区分两个内容区域

### 热门商品（瀑布流）
- **布局**：两列瀑布流（masonry），不等高，顶部对齐，图片高度随商品图自然撑开
- **标题**：`热门商品`
- **卡片内容**：
  - 商品图片（不等高，随图片比例）
  - 商品名称
  - 价格 + 加购按钮（绿色圆角方块 `+`）
  - 企业来源标签（底部分隔线下方，`🌿 绿源农场 >`，可点击跳转企业页）
- **无限滚动**：上拉加载更多，底部加载提示

## 企业 Tab

### 筛选标签横滑
- 筛选维度（与商品 Tab 不同）：全部 / 🌿 有机认证 / 🍎 水果 / 🍵 茶叶 / 📍 附近
- 样式与商品 Tab 分类标签一致

### 企业卡片列表
- **布局**：单列全宽卡片，纵向列表滚动
- **卡片内容**：
  - 头部行：企业 Logo（48px 圆角方块）+ 企业名称 + 认证标签（如"有机认证"）
  - 副信息：地区 · 距离 · 好评率
  - 热门商品缩略图行：3 个商品小卡片（图片 + 名称 + 价格）+ `+N` 更多提示
- **交叉引流**：点击商品缩略图跳转商品详情，点击卡片其他区域跳转企业详情页
- **无限滚动**：上拉加载更多

## 地图模式

点击标题行的地图切换按钮进入。

### 布局
- **全屏地图**：地图占满屏幕
- **浮动元素**（叠加在地图上方）：
  - 标题行：`发现` + 列表切换按钮 + 购物车（半透明白色背景）
  - 搜索框：浮动在地图上，白色背景 + 阴影
  - Tab 标签：浮动胶囊式按钮（`商品` / `企业`），选中态绿色填充

### 标点行为
- **企业 Tab**：标点 = 企业位置
- **商品 Tab**：标点 = 产地/发货地
- 用户位置蓝点 + 范围圈

### 点击标点交互
- 底部弹出浮动企业卡片：
  - 企业 Logo + 名称 + 认证标签
  - 距离信息
  - 热门商品缩略图（3个 + `+N`）
  - "进店 →" 按钮

## 交叉引流设计

解决方案 A（Tab 切换）中商品与企业割裂的问题：

| 从 → 到 | 实现方式 |
|----------|----------|
| 商品 → 企业 | 商品卡片底部企业来源标签，点击跳转企业详情页 |
| 企业 → 商品 | 企业卡片热门商品缩略图，点击跳转商品详情页 |

## 与现有实现的差异

| 项目 | 现在 | 改为 |
|------|------|------|
| 商品/企业布局 | 线性堆叠（AI推荐→企业横滑→热门商品） | Tab 切换，各自独立 |
| 企业展示 | 横滑卡片夹在商品中间，6 个 | 独立 Tab，全宽卡片列表，无限滚动 |
| AI 推荐区 | 两列网格等高 | 横向滑动等高卡片 |
| 热门商品 | 两列等高网格 | 瀑布流不等高 |
| 商品卡片 | 无企业信息 | 底部增加企业来源标签 |
| 企业卡片 | 简单信息卡 | 增加热门商品缩略图 |
| 地图模式 | 半屏地图 + 地图商选择 | 全屏地图 + 浮动控件 + 底部弹出卡片 |
| 搜索框 | 带提示文案 | 简洁搜索框，无 AI 标识/麦克风 |

## 技术考量

### 瀑布流实现
- React Native 中 FlashList 不原生支持瀑布流
- 方案：使用 `@shopify/flash-list` 的 `MasonryFlashList` 或手动双列布局（两个独立 FlatList 交替分配数据）
- 需要提前获取图片高度或使用 `onLayout` 动态调整

### Tab 切换
- 使用 React 状态管理 Tab 切换，保持两个 Tab 的滚动位置独立
- 企业 Tab 数据独立查询，带分页

### 地图模式浮动控件
- 使用 `position: absolute` 将搜索框和 Tab 叠加在地图组件上方
- 底部企业卡片使用 `BottomSheet` 或自定义 Animated 组件

### 企业热门商品数据
- 需要后端接口支持：企业列表接口返回每个企业的 top 3 热门商品（含缩略图、名称、价格）
- 或使用现有企业详情接口的商品字段
