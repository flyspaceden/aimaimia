# Codex Prompt：构建「爱买买」React Native + Expo（仅前端）高质量 App

你是**资深移动端工程师 + 产品设计师**。请在一个全新的 **Expo（Managed）+ TypeScript** 项目中，实现一个达到“优秀商业 App 设计水平”的移动端前端应用：**爱买买 - AI 赋能农业电商平台**。

> 重要：**只做前端**。需要有“后端接口的形态”，但**不要实现/枚举每一个后端接口**。请使用 **Mock 数据 + Repository 抽象层**模拟数据来源，确保未来接入真实后端时 UI 与业务层无需大改。

---

## 1. 目标与范围

### 1.1 产品定位
- 现代农业 + 科技感（AI 赋能）+ 清新自然的品牌视觉
- 电商交易 + 企业可信展示（数字展览馆）+ 社区互动（心愿池、爱买买圈）+ 个人中心（资产/订单/会员）

### 1.2 必须实现（前端）
- 底部 **5 Tab**：**首页 / 数字展览馆 / 心愿池 / 爱买买圈 / 我的**
- 关键详情页：商品详情、企业详情、心愿详情、帖子详情
- 典型电商体验：瀑布/双列商品流、加购动效、购物车角标、订单列表（可用 mock）
- “可信”内容体验：企业资质/检测报告（做 UI + 预览占位）
- “AI”入口占位：推荐理由、AI 农管家（对话 UI 占位）

### 1.3 非目标（本次不做）
- 不实现真实后端、支付闭环、短信登录等（仅保留 UI/流程占位）
- 不做复杂权限与风控（但要有 UI 入口与状态占位）

## 2. 技术栈与工程要求（必须遵守）

### 2.1 技术栈（Expo Managed）
- React Native + Expo（最新稳定 SDK）
- TypeScript
- 路由：`expo-router`
- 网络/缓存与数据流：`@tanstack/react-query`（用来管理 mock repo 的异步数据）
- 全局状态：`zustand`（购物车、用户信息、偏好）
- 表单：`react-hook-form` + `zod`
- 图片：优先用 `expo-image`（列表性能）
- 动画：尽量使用 RN 原生 `LayoutAnimation` 或 `react-native-reanimated`（如 Expo 默认可用）实现轻量微交互（加购、切换、卡片反馈）
- UI 不依赖大型 UI 框架（避免 Paper/Tamagui 全量引入），请用**自研 Design Tokens + 组件库**实现高级质感与一致性

### 2.2 工程结构（必须清晰可维护）
- 使用 **src/** 目录组织：`components/`, `screens/`（或 `app/` 路由内拆分), `features/`, `store/`, `repos/`, `mocks/`, `theme/`, `utils/`
- Repository 模式（示例）：`ProductRepo`, `CompanyRepo`, `WishRepo`, `FeedRepo`, `OrderRepo`, `UserRepo`
  - **Repo 对外暴露方法**（如 `list`, `getById`），内部目前返回 mock 数据（可用 `setTimeout` 模拟延迟）
  - UI 层通过 React Query 调用 Repo；未来替换为真实 API 时，只改 Repo 实现

### 2.3 质量标准（必须做到）
- 每个列表页必须有：**Skeleton / Empty State / Error State**
- 可访问性：按钮可点击区域足够大；文字对比度合理；支持系统字体缩放（不强行禁用）
- 性能：首页商品流/爱买买圈信息流必须使用 `FlatList`，优化 `keyExtractor`、`initialNumToRender`、`windowSize`
- 统一的状态提示：Toast、加载中、错误提示（不要每个页面各写一套）

## 3. 信息架构与页面清单（必须实现这些页面）

### 3.1 Tab（底部 5 个）
1) **首页 Home**
- 顶部：Logo + 搜索（可跳转 Search）+ 消息/购物车入口（角标）
- Banner 轮播（mock 图片）
- 分类入口（生鲜/蔬菜/水果/粮油等，可 mock）
- 商品瀑布/双列卡片流（图片、标题、价格、产地、可信标签）
- 商品卡支持：收藏（可 mock）、加购（有动效）

2) **数字展览馆 Museum**
- 企业列表（默认列表）+ 筛选条（地区/品类/认证类型，仅 UI）
- 企业卡：封面、名称、主营、认证徽章、距离（可 mock）
- 企业详情页（见 3.2）

3) **心愿池 Wishes**
- 双 Tab：发表心愿 / 发现心愿
- 发现页：瀑布/列表，支持排序：热门/最新/快实现（UI）
- 心愿详情页（见 3.2）
- 发布心愿页（表单 + 标签 + 可选 @企业）

4) **爱买买圈 Circle**
- 顶部三 Tab：推荐 / 关注 / 企业
- 信息流卡片（图片/视频占位也可）
- 帖子可挂商品：卡片底部展示商品标签，点击弹出商品小卡（Bottom Sheet）
- 发布入口（中间凸起按钮或右下浮动按钮）：选择模板（产品故事/种植日志/食谱）

5) **我的 Me**
- 身份名片：头像、昵称、欢迎语、会员等级进度条（mock）
- 任务与资产中心：任务列表（mock）、签到卡（UI）
- 订单入口：待付款/待发货/待收货角标；订单列表页
- AI 农管家入口（聊天 UI 占位）
- 设置/帮助/隐私政策入口（静态页）

### 3.2 关键详情页（栈路由）
- **商品详情** `/product/[id]`
  - 大图轮播、价格、产地、可信标签、图文详情
  - 固定底部 CTA：加入购物车 / 立即购买（购买仅占位提示）
- **企业详情** `/company/[id]`
  - 顶部海报 + 企业名 + 可信摘要条（认证数、最新检测日期、主营品类）
  - 内部分段 Tab（Segmented Control）：档案 / 资质 / 检测报告 / 实时风采 / 日历预约 / 组团看板
  - 检测报告：卡片列表 + “预览”占位（可用 WebView 打开 pdf url 占位）
  - 日历预约：列表/日历占位 + 底部抽屉表单
  - 组团看板：进度条 + 倒计时 + 一键参团（占位）
- **心愿详情** `/wish/[id]`
  - 内容 + 标签 + 状态（已采纳/规划中/已实现）+ 进度条/里程碑时间线
  - 官方/企业回复区 + 评论区（mock）
- **帖子详情** `/post/[id]`
  - 正文 + 图片
  - 关联商品标签（可弹商品小卡）
  - 评论（mock）

### 3.3 其它辅助页
- 搜索页 `/search`：搜索框 + 热门推荐 + 结果列表（mock）
- 购物车 `/cart`：商品列表、数量修改、合计、结算按钮（占位）
- 订单列表 `/orders` 与订单详情 `/orders/[id]`（mock）
- 静态页：设置、隐私、关于
